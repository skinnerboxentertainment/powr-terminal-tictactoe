# AutoMagically Assets and Build

This skill defines how assets are loaded, bundled, and configured for build.

---

## 1. Asset Loading with PixiJS `Assets`

Use PixiJS v8 `Assets` class for all game assets:

```typescript
import { Assets } from "pixi.js"

// Initialize with a manifest
await Assets.init({
  manifest: "assets/manifest.json",
})

// Load a bundle per scene
const assets = await Assets.loadBundle("game-scene")
// assets contains textures, spritesheet data, etc.
```

Rules:
- Load **bundles** per scene, not individual textures in `update()`.
- Preload the first scene's bundle during the boot screen.
- Use `Assets.loadBundle("scene-name")` in `scene.enter()`.
- Do not use PixiJS v7 `Loader` class (removed in v8).

---

## 2. Asset Bundles by Scene

```
assets/manifest.json
assets/gameplay/     ← sprites, tilemaps for gameplay
assets/ui/           ← button sprites, HUD elements
assets/audio/        ← music, SFX, sprites, ambience
```

Each scene gets its own bundle key in the manifest. A shared `core` bundle
contains assets used by multiple scenes (fonts, common UI, player sprites).

---

## 3. Audio Codec Strategy

Provide two formats per audio file:

```
assets/audio/music/bg-theme.mp3     ← primary (smaller, widely supported)
assets/audio/music/bg-theme.webm    ← fallback (Opus codec, better quality)
```

Howler picks the first playable format from the `src` array. WebM with Opus
is the best quality-to-size ratio. MP3 is the universal fallback.

---

## 4. Imports vs `public/`

```
import textureUrl from "../assets/gameplay/player.png"
```

TypeScript/Vite handles hashing and bundling. Use imports for textures,
spritesheets, fonts, and data files that Vite can process.

Place in `public/` only files that must be served as-is with a fixed URL:
- `manifest.json` (referenced by `Assets.init`)
- `favicon.ico`
- HTML files (if any)

---

## 5. Spritesheet Convention

Use PixiJS spritesheets for any sprite with multiple frames or variants:

```
assets/gameplay/player/
  player.png              ← spritesheet texture atlas
  player.json             ← PixiJS spritesheet data (generated by tools)
```

Load with `Assets.load` using the `.json` path — PixiJS automatically reads
the texture atlas.

---

## 6. Font Loading

Load fonts before the first scene that uses text:

```typescript
// In boot scene
await Assets.loadBundle("fonts")
```

Store fonts in `assets/ui/fonts/`. Supported formats: `.woff2` (preferred),
`.ttf` (fallback). Preload in the boot/loading scene, not per-gameplay scene.

---

## 7. Build Configuration

Vite settings: already configured in `vite.config.ts`. Do not change unless
adding new asset types or modifying output paths.

Key defaults:
- Build output: `dist/`
- Target: `es2022`
- Asset hashing: enabled (Vite default)
- Base path: `/` (change via `--base` flag for subdirectory deployment)

---

## 8. Anti-Patterns

- PixiJS v7 `Loader` API (`app.loader` — removed in v8).
- `Assets.load` inside `update()` — load in `enter()`, cache in scene.
- Importing textures from `public/` (they won't be hashed).
- Missing WebM fallback for audio files (MP3-only may not play in all browsers).
- Loading the entire asset manifest at boot (load bundles per scene).
